# Day 5 Testing Guide: Public Share Page

## What Was Built

Public Share Page feature allows users to make Agent Runs publicly accessible via shareable links. Perfect for hackathon submissions, DevRel demos, and audit reviews.

## Modified Files

### New Files Created
1. `src/lib/public-id.ts` - Public ID generator utility
2. `src/components/share-panel.tsx` - Share controls component
3. `src/app/trace/[publicId]/page.tsx` - Public trace route
4. `src/app/trace/[publicId]/public-trace-client.tsx` - Public report page

### Modified Files
1. `src/types/run.ts` - Added `is_public`, `public_id`, `published_at` fields
2. `src/lib/storage.ts` - Added `updateRunPublicStatusBrowser()` and `getPublicRunBrowser()`
3. `src/app/runs/[id]/run-detail-client.tsx` - Added SharePanel integration
4. `README.md` - Added Day 5 documentation
5. `BUILD_LOG.md` - Added Day 5 build log

## How to Test

### 1. Make a Run Public

1. Start dev server (already running at http://localhost:3000)
2. Navigate to Dashboard: http://localhost:3000/dashboard
3. If no projects exist, click "Load demo project"
4. Click on a project to view project detail
5. Click on any run to view run detail
6. Scroll down to "Share Proof-of-Execution" section
7. Click "Make Public" button
8. Observe:
   - Button changes to show public link
   - Public link format: `http://localhost:3000/trace/trace_<uuid>`
   - Copy Link button appears
   - View Public Page button appears
   - Unpublish button appears

### 2. Copy Public Link

1. After making run public, click "Copy Link" button
2. Observe:
   - Button shows check icon for 2 seconds
   - Link is copied to clipboard
3. Paste link in a new browser tab or incognito window

### 3. View Public Page

1. Click "View Public Page" button, OR
2. Paste the copied link in browser, OR
3. Open in incognito window (to simulate unauthenticated access)
4. Observe public report displays:
   - AgentTrace header with "Public Report" badge
   - Run title and status badges
   - Published date
   - Executive Summary (if AI summary was generated)
   - Project Context (name, agent, chain, wallet, GitHub, demo)
   - Execution Timeline (8 steps)
   - MCP/Tool Call Evidence (if metadata exists)
   - Transaction Context (if transaction-to-trace run)
   - AI Audit Report (if generated)
   - Task Receipt (hash and JSON)
   - Footer: "This proof-of-execution report was generated by AgentTrace"

### 4. Test Unpublish

1. Return to run detail page (authenticated)
2. Click "Unpublish" button
3. Confirm dialog
4. Observe:
   - Section reverts to "Make Public" button
   - Public link is removed
5. Try to access the public link again
6. Observe:
   - "Report Not Found" page displays
   - Error message: "This proof-of-execution report does not exist or is no longer publicly accessible"

### 5. Test Invalid Public ID

1. Navigate to: http://localhost:3000/trace/invalid_id_123
2. Observe:
   - "Report Not Found" page displays
   - No white screen or error

### 6. Test Refresh Persistence

1. Make a run public
2. Copy the public link
3. Refresh the run detail page
4. Observe:
   - Public link still displays
   - Same public_id
5. Refresh the public trace page
6. Observe:
   - Report still loads correctly

### 7. Test Backward Compatibility

1. Create a new manual run (without making it public)
2. Navigate to run detail page
3. Observe:
   - Share panel displays with "Make Public" button
   - No errors
4. Old runs in localStorage still load correctly

## Current Limitations

1. **No Supabase Schema Migration**: Day 5 focuses on Dev Mode (localStorage). Supabase tables don't have `is_public`, `public_id`, `published_at` columns yet. When Dev Mode is off, public share won't work until schema is updated.

2. **No Analytics**: No view count or access tracking for public links.

3. **No Expiration**: Public links don't expire automatically. User must manually unpublish.

4. **No Social Preview**: Public pages don't have OpenGraph/Twitter Card meta tags for rich link previews.

5. **No PDF Export**: Public reports can't be downloaded as PDF.

6. **No Custom Domains**: Public links use the same domain as the app (`localhost:3000` in dev, your domain in production).

7. **No Embeddable Widgets**: Can't embed public trace in iframe or as a widget.

8. **No Permission Granularity**: It's binary (public or private). No "unlisted" or "password-protected" options.

## What to Polish Next

### High Priority
1. **Supabase Schema Migration**: Add columns to `runs` table with RLS policies
2. **OpenGraph Tags**: Add meta tags to public trace page for link previews
3. **Mobile Optimization**: Test and improve mobile layout for public reports

### Medium Priority
4. **Analytics**: Track view counts and referrers for public links
5. **Expiration Options**: "Make public for 7 days" or "expires at hackathon end"
6. **PDF Export**: Generate downloadable PDF report from public page
7. **Embed Codes**: Generate iframe embed codes for documentation sites

### Low Priority
8. **Custom Slugs**: Allow users to customize public_id (e.g., `trace_my-demo-run`)
9. **Password Protection**: "Make public with password" option
10. **QR Codes**: Generate QR code for public link (useful for posters/presentations)
