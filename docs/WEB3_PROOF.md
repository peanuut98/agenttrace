# Web3 Proof Anchoring

AgentTrace generates an off-chain Proof-of-Execution Report for every Web3 AI Agent run. To make those reports independently referenceable, AgentTrace ships a minimal on-chain registry contract that anchors the off-chain receipt hash to a public testnet event log.

## Scope

The on-chain piece is intentionally narrow:

- It **does not** automatically scrape on-chain transactions. Transaction context is gathered via the Transaction-to-Trace flow / Explorer API integration.
- It **does not** automatically analyse what an agent did. AI Audit Reports are generated by the configured AI provider (e.g. Z.ai GLM-5.1).
- It **does not** sign anything on the user's behalf. Every `registerProof()` call requires manual wallet confirmation.
- It **does not** hold or transfer funds. The contract is event-emit only.

The contract is therefore an append-only public index that maps `bytes32 receiptHash` → `(submitter, publicReportUrl, timestamp)` via emitted events. Anyone holding a receipt JSON can recompute its hash and search BaseScan for the matching `ProofRegistered` event.

## How the on-chain registration works

1. The user clicks **Register Proof On-chain** on a Public Report (only visible in dev mode and only when `NEXT_PUBLIC_PROOF_REGISTRY_ADDRESS` is set and a browser wallet is detected).
2. The wallet (e.g. MetaMask) prompts to connect.
3. The wallet is asked to switch to **Base Sepolia (chainId 84532)**. If the chain is not added, the user is prompted to add it.
4. AgentTrace ABI-encodes a `registerProof(bytes32 receiptHash, string publicReportUrl)` call.
5. The wallet shows a `Confirm transaction` dialog. The user manually signs.
6. The submitted transaction hash is persisted alongside the receipt and rendered on the Public Report with a BaseScan link.

At no point does AgentTrace see, store, or transmit the user's private key.

## Safety boundaries

- AgentTrace **does not** hold private keys.
- This action **requires manual wallet confirmation**.
- The contract **does not transfer funds** and has no payable entrypoint.
- This **only anchors the receipt hash** on-chain — it does not move tokens or grant permissions.
- **Testnet only** for demos. Re-audit before any mainnet deployment.

## Verifying a registration on-chain

Anyone can verify a registered proof without trusting AgentTrace:

1. Visit `https://sepolia.basescan.org/address/<CONTRACT_ADDRESS>#events`.
2. Filter by the `ProofRegistered` event topic.
3. Match the `receiptHash` topic against the SHA-256 hash of the canonical receipt JSON published in the Public Report.
4. Inspect `submitter` and `timestamp` for who anchored it and when.
